使い方

　ダウンロードしていただいたファイルを解凍すると、以下のファイルが出てきます。
○1 MoveMouseCursorPlugin 説明書.html (このファイル)
○2 MoveMouseCursorPlugin.tjs (プラグイン本体)
　first.ks の @iscript 中に、
@iscript
　　Scripts.execStorage("MoveMouseCursorPlugin.tjs");
@endscript
と書いてください。 MoveMouseCursorPlugin が読み込まれます。
　これで下準備は完了です。

タグ名

  • @movecursor
  • @waitmovecursorstop

    • @movecursor

    　マウスカーソルの移動を行います。
    x : 移動後のX座標です。画面左上端が 0 で、右にいくほど大きくなります
    y : 移動後のY座標です。画面左上端が 0 で、下にいくほど大きくなります
    time : 移動する時間を設定します。大きな値を設定するとゆっくりと動きます
    accel : 加速度を持った移動を行うかどうかを設定します。2 以上を指定すると徐々に動きが速く、-2 以下だと徐々に動きは遅くなります。
    force : trueを指定すると、timeで設定した時間が経つまでユーザはマウスを動かせません。
    maxpermit : ユーザーがマウスを移動した際に、移動量の X、Y 合計がこの値以上だと実行を停止します

    @waitmovecursorstop

    　移動が終わるのを待ちます。略記として @wmc で記述も可能です。
    canskip : スキップ可能か

    使用例

    　@MoveCursor x=80 y=60 time=1000 accel=-5
    　@WaitMoveCursorStop
    　1秒間かけて(80,60)へと移動します。accel が -5 なので、視覚的には初速が最大でブレーキを掛けながら進んでいる感じです。

    　@MoveCursor x=80 y=60 time=1000 accel=-5
    　@MoveCursor x=640 y=480 time=2000
    　@WaitMoveCursorStop
    2 秒間かけて(640,480)へと移動します(標準サイズの画面では右下)。
    ひとつめの @MoveCursor タグの直後にまた @MoveCursor があるので、ひとつめは実行されません。

    　@MoveCursor x=80 y=60 time=1000 force=true
    　@WaitMoveCursorStop
    　forceにtrueを指定しているので、1000ミリ秒経つまでユーザはマウスを動かすことができません。

    注意事項

    　0.5X より KAG プラグイン部分とマウスを動かす部分を切り離したので、TJS からマウス移動を呼ぶことが出来ます。
    　例えば ButtonLayer 内で
    MouseCursorMover.set(%[layer:this, x:width/2, y:height/2, time:300, accel:-4]);
    　と書くと、マウスがボタンの中央に移動します。
    　ウィンドウ内の座標ではなく layer に指定したレイヤに対する座標を取っているので、ウィンドウ内でのボタンの位置は関係有りません。
    　オブジェクトの名前は MouseCursorMover で、set は辞書配列をとります。layer を指定しないと動かないので注意して下さい。

    　タグ実行時には背景レイヤの位置を取得してそれを元に動かしているので、@quake タグ使用時にマウスも一緒に揺れるかも（未検証）

    　所定位置まで移動する、もしくは移動中にユーザーがマウスを動かした場合に実行を停止する機能が付いています。(0.4X より)

    　吉里吉里 2.32 以降での動作確認をしています。明記されている場合を除き、それ以前のバージョンでの動作は保証しません。
